Scrapy加playwright爬虫框架

发表于2026-05-19|更新于2026-05-19|教程
|浏览量:

架构概览

Scrapy (/ˈskreɪpaɪ/) 是一个用于爬取网站并提取结构化数据的应用框架，可用于各种有用的应用，如数据挖掘、信息处理或历史存档。

Scrapy 的主要优势之一：请求是异步调度和处理的。这意味着 Scrapy 不需要等待一个请求完成并被处理，它可以在此期间发送另一个请求或做其他事情。这也意味着即使某个请求失败或处理时发生错误，其他请求仍可以继续进行。  

概览

下图展示了 Scrapy 架构的概览及其组件，并概述了系统内部发生的数据流（由红色箭头表示）。下面包含对各组件的简要说明，以及指向其详细信息的链接。数据流也在下方进行了描述。

数据流

Scrapy 中的数据流由执行引擎控制，流程如下：

  1. 引擎从爬虫获取初始抓取请求（Requests）。
  2. 引擎将请求放入调度器中进行调度，并请求下一个要抓取的请求。
  3. 调度器将下一个请求返回给引擎。
  4. 引擎将请求发送到下载器，途中经过下载器中间件（参见 process_request()）。
  5. 页面下载完成后，下载器会生成一个响应（Response）（包含该页面），并将其发送给引擎，途中经过下载器中间件（参见 process_response()）。
  6. 引擎从下载器接收响应，并将其发送给爬虫进行处理，途中经过爬虫中间件（参见 process_spider_input()）。
  7. 爬虫处理响应并向引擎返回提取到的项目（Items）和新的请求（Requests）（用于后续跟踪），途中经过爬虫中间件（参见 process_spider_output()）。
  8. 引擎将处理后的项目发送到项目管道，然后将处理后的请求发送到调度器并请求下一个可能的抓取请求。
  9. 流程重复（从步骤 3 开始），直到调度器中不再有请求。

使用选择器

Scrapy 自带了一套提取数据的机制。它们被称为选择器（selectors），因为它们通过 XPath 或 CSS 表达式来“选择” HTML 文档的特定部分。

XPath 是一种在 XML 文档中选择节点的语言，也可以用于 HTML。CSS 是一种为 HTML 文档应用样式的语言。它定义了选择器，用于将这些样式与特定的 HTML 元素关联起来。

Xpath选择器

在 XPath 表达式中

// 从当前节点（或文档根）开始，在整个文档中递归地查找所有匹配的后代元素（包括子元素、孙元素等，任意深度）

✅ 返回结果是：'第一个'
因为 //span 找到所有 <span>（共3个），.get() 取第一个。

结论：

  • //span 匹配所有 <span> 标签；
  • .get() 默认返回第一个匹配项的值,getall()返回全部；
  • 若需指定第 N 个，应使用 (//span)[N]

通过属性找标签要：标签名[@属性名=xxx]

CSS选择器

根据 W3C 标准，CSS 选择器 不支持选择文本节点或属性值。但在网页爬取上下文中，选择这些内容非常重要，因此 Scrapy (parsel) 实现了一些 非标准伪元素：

  • 要选择文本节点，请使用 ::text
  • 要选择属性值，请使用 ::attr(name)，其中 name 是你想要获取值的属性名称
